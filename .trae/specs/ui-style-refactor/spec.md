# 中医药管理系统 - UI 风格重构 PRD

## Overview
- **Summary**: 将现有的中医药管理系统前端 UI 风格重构为设计稿中的国风设计，在保持所有功能完整不变的前提下，提升视觉体验和用户感受
- **Purpose**: 解决现有界面设计过于简单朴素的问题，采用设计稿中的国风配色和布局，打造更具中医药文化特色的现代化界面
- **Target Users**: 中医药管理系统的所有使用者，包括管理员、药师、医生等

## Goals
- 将设计稿中的国风配色方案应用到现有 Vue 项目
- 保持所有现有功能完全不变（登录、权限管理、中药管理、药方管理、库存管理、仪表板）
- 保持中英文双语切换功能完整
- 优化界面布局和视觉层次
- 提升用户体验和界面美观度

## Non-Goals (Out of Scope)
- 不修改后端 API 和数据结构
- 不添加新的功能模块
- 不修改现有的业务逻辑
- 不重构代码架构，只调整样式和布局

## Background & Context
- **现有技术栈**: Vue 3 + Composition API + Vue Router + Pinia + Vite
- **设计稿技术栈**: React + TypeScript + Tailwind CSS + shadcn/ui
- **设计稿特点**: 国风配色（墨绿色、赭石色、朱红色）、清晰的布局层次、现代化组件设计
- **需要移植的核心元素**: 配色方案、布局结构、卡片样式、按钮样式、表格样式

## Functional Requirements
- **FR-1**: 应用国风配色方案到整个前端界面
- **FR-2**: 重构 AppLayout 组件，采用设计稿的侧边栏和顶部导航布局
- **FR-3**: 重构所有页面的卡片、按钮、表格等组件样式
- **FR-4**: 保持所有现有页面功能完全正常（登录、仪表板、用户管理、中药管理、药方管理、库存管理）
- **FR-5**: 保持中英文双语切换功能完整可用

## Non-Functional Requirements
- **NFR-1**: 页面加载速度不受影响（与现有版本相当）
- **NFR-2**: 响应式设计，支持不同屏幕尺寸
- **NFR-3**: 界面风格统一一致，符合设计稿的国风美学
- **NFR-4**: 无障碍访问性保持或提升

## Constraints
- **Technical**: 必须使用现有的 Vue 3 技术栈，不能引入 React 或其他新框架
- **Business**: 必须在保持功能不变的前提下进行样式重构
- **Dependencies**: 保留现有所有依赖，可考虑引入 Tailwind CSS 用于样式管理

## Assumptions
- 设计稿中的所有颜色值可以正确应用到 CSS
- 现有的组件结构不需要大幅重构
- 用户会接受界面风格的变化
- 所有现有测试在样式重构后仍然通过

## Acceptance Criteria

### AC-1: 配色方案正确应用
- **Given**: 用户打开应用
- **When**: 查看任何页面
- **Then**: 页面使用设计稿的国风配色（墨绿 #2D6A4F、赭石 #D9B38C、朱红 #E63946、浅青 #457B9D）
- **Verification**: `human-judgment`
- **Notes**: 检查主要按钮、卡片、侧边栏等关键元素的颜色

### AC-2: 侧边栏布局正确
- **Given**: 用户已登录
- **When**: 查看主界面
- **Then**: 左侧显示设计稿风格的墨绿色侧边栏，包含菜单导航
- **Verification**: `human-judgment`
- **Notes**: 侧边栏宽度约 220px，可考虑支持折叠功能

### AC-3: 所有功能保持正常
- **Given**: 用户使用系统的任何功能
- **When**: 执行登录、CRUD 操作、搜索筛选等
- **Then**: 所有功能行为与重构前完全一致
- **Verification**: `programmatic`
- **Notes**: 包括表单提交、数据加载、权限控制等

### AC-4: 中英文切换正常
- **Given**: 用户点击语言切换器
- **When**: 在中文和英文之间切换
- **Then**: 所有界面文本正确显示对应语言
- **Verification**: `human-judgment`
- **Notes**: 检查所有页面的翻译是否正常

### AC-5: 响应式布局正常
- **Given**: 用户使用不同设备访问
- **When**: 在桌面、平板、手机等不同屏幕尺寸查看
- **Then**: 界面布局合理响应，内容可正常访问
- **Verification**: `human-judgment`
